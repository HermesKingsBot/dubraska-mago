# SQLite ŌåÆ PostgreSQL (Neon) Migration

## Complete Migration Workflow

### 1. Update Schema

```prisma
// prisma/schema.prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

No schema changes needed for field types ŌĆö `String`, `Boolean`, `Int`, `Float`, `DateTime` all map correctly. JSON arrays stored as `String` work in both SQLite and PostgreSQL.

### 2. Clean Old SQLite Artifacts

```bash
rm -rf prisma/migrations
rm -f prisma/dev.db
```

### 3. Create Migration for PostgreSQL

```bash
mkdir -p prisma/migrations/000001_init

cat > prisma/migrations/migration_lock.toml << 'EOF'
# Please do not edit this file manually
provider = "postgresql"
EOF

npx prisma migrate diff --from-empty --to-schema-datamodel prisma/schema.prisma --script > prisma/migrations/000001_init/migration.sql
```

### 4. Apply to Neon

```bash
DATABASE_URL="postgresql://..." npx prisma migrate deploy
npx prisma generate
```

### 5. Configure Vercel

```bash
source .env.vercel
DATABASE_URL=$(grep "^DATABASE_URL=" .env | cut -d'"' -f2)

# Production environment (Sensitive type)
echo "$DATABASE_URL" | npx vercel --token "$VERCEL_TOKEN" env add DATABASE_URL production
```

**Note:** Preview env vars require interactive branch input ŌĆö configure via dashboard if needed.

### 6. Update package.json

```json
{
  "scripts": {
    "postinstall": "prisma generate",
    "db:migrate": "prisma migrate deploy",
    "db:push": "prisma db push",
    "db:seed": "tsx prisma/seed.ts"
  }
}
```

### 7. Seed Data

```bash
DATABASE_URL="postgresql://..." npx tsx prisma/seed.ts
```

## Neon-Specific Notes

- **Connection string**: `postgresql://user:***@host:5432/db?sslmode=require`
- **Pooled connections**: Add `pgbouncer=true` if using connection pooling
- **Serverless**: Neon scales to zero ŌĆö first request may have cold start latency
- **No `pgbouncer=true` needed** for basic Prisma setup (Prisma handles pooling internally)
- **Edgecompatible**: Works with Vercel Edge functions via ` PrismaAccelerate` or direct connections

## Post-Migration Verification

```bash
# Check all tables exist
curl -s "https://your-site.vercel.app/api/products" | python3 -m json.tool

# Verify data is served
curl -s "https://your-site.vercel.app/" | grep "Nuestras piezas"
```

## Common Issues

1. **`prisma migrate deploy` fails with "database does not exist"** ŌĆö Run on a fresh Neon database (already created from dashboard)
2. **`prisma migrate diff` shows `--to` unknown option** ŌĆö Use `--to-schema-datamodel` not `--to`
3. **`write_file` tool redacts secrets with `***`** ŌĆö Use `terminal` with `sed` to write real connection strings to `.env`
4. **Vercel env var type** ŌĆö Use `Sensitive` type for DATABASE_URL (hides value in UI)
